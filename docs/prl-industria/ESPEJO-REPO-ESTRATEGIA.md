# Estrategia de Repositorio Espejo (Modelo Mixto)

## Objetivo

Mantener una fuente de verdad en este proyecto y publicar un repo espejo documental para consumo operativo de equipos PRL.

## Principio

- Fuente de verdad: carpeta docs/prl-industria en este proyecto.
- Espejo: repositorio documental separado con contenido validado para operacion.

## Que se espeja

1. Documentos de flujo.
2. Plantillas operativas.
3. Matriz de obligaciones.
4. Plan de implementacion y tablero.

## Que no se espeja

1. Artefactos de build frontend.
2. Codigo de la app no necesario para operacion documental.
3. Borradores no aprobados.

## Cadencia de sincronizacion

- Minimo semanal.
- Extraordinaria ante cambio legal relevante o incidente critico.

## Criterio de publicacion al espejo

1. Documento con base legal explicita.
2. Plantilla validada por owner PRL.
3. Control de cambios actualizado.

## Ownership

- Aprobacion legal/tecnica: Responsable PRL.
- Aprobacion operativa: Supervisor principal.
- Publicacion espejo: Owner documental.
